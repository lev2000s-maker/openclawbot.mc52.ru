# metalbot-admin

Панель администрирования (PHP 8+, SQLite) для размещения на shared-хостинге, например reg.ru, поддомен `https://bot.mc52.ru/`, корень веб-пространства: `/www/bot.mc52.ru/`, точка входа `index.php`.

**Принцип:** сначала интерфейс, данные, API; без «сверх-архитектуры». Локальный рабочий скелет — **до** адаптации под хостинг.

---

## Критически важный порядок разработки

1. **Сначала** сделать и **проверить проект локально**.
2. **Только потом** адаптировать под reg.ru (или иной shared-хостинг).

### Этап 1. Локальный запуск

1. Скопировать `config.local.php.example` в `config.local.php` (опционально; без файла подставляются встроенные defaults — для разработки достаточно, для прод — задать `api_key` и `app_secret`).
2. Создать БД:  
   `php db/seed.php`  
   (повторно без флага не перетирает пользователя; полный сброс: `php db/seed.php --reset` — **удаляет** `database.sqlite`).
3. Поднять встроенный сервер **из корня** проекта:  
   `php -S localhost:8000`
4. Открыть: `http://localhost:8000/`
5. Войти: **admin** / **admin1234** (смените пароль / пересоберите seed на проде).
6. Проверить: обзор, поставщики, номенклатура, загрузка прайса, разделы лидов/задач/сценариев/промтов/настроек.
7. API: `http://localhost:8000/api/health.php` (без ключа), остальные `api/*.php` — с заголовком `X-Api-Key: <ваш_ключ_из_конфига>` (или `Authorization: Bearer <ключ>`) либо из браузера **в той же сессии** после входа.

### Локальный smoke-test (чек-лист)

- [ ] `php -S localhost:8000` в каталоге `metalbot-admin`
- [ ] `http://localhost:8000` открывается
- [ ] Вход в панель работает
- [ ] `dashboard` открывается
- [ ] Поставщик создаётся и снова виден в списке
- [ ] Позиция номенклатуры создаётся и видна
- [ ] Прайс загружается (раздел «Прайсы»)
- [ ] `GET /api/health.php` — JSON
- [ ] `GET /api/suppliers.php` с заголовком `X-Api-Key` — JSON со списком
- [ ] `db/database.sqlite` существует и меняется после операций (можно открыть sqlite3, DB Browser, и т. п.)

**Главное правило: локально работает → потом reg.ru. Не начинать с хостинга.**

## Этап 2. Адаптация под reg.ru

**Только** после успешного локального smoke-test:

1. Залить содержимое `metalbot-admin/` в `/www/bot.mc52.ru/` (или подпапку — тогда в `config.local.php` задать `base_path` с ведущим слешем, например `'/app'`).
2. Убедиться, что Apache обрабатывает `.htaccess` (обычно включено; для подпапок может понадобиться `RewriteBase` в `.htaccess`).
3. **Права (ориентир):** каталоги `db/`, `uploads/`, `logs/` — **запись** для веб-пользователя (часто `0755` для каталогов, файлы `0644`); `db/database.sqlite` — запись (например `0644` или `0664` в зависимости от umask).
4. Создать/обновить БД **на сервере**: `php db/seed.php` по SSH **или** один раз залить подготовленный `database.sqlite` + при необходимости запустить seed только для миграций.
5. Скопировать `config.local.php` с **уникальными** `app_secret` и `api_key`.
6. Проверить: `https://bot.mc52.ru/`, логин, CRUD, загрузка файла, `https://bot.mc52.ru/api/health.php`.

### Защита каталогов

- `db/.htaccess`, `logs/.htaccess`, `uploads/.htaccess` — запрещают/ограничивают прямой доступ; основная копия БД **не** должна быть в открытом `public_html` без защиты.
- SQLite-файл лежит в `db/database.sqlite` (см. `db_path` в конфиге); прямой заход `https://.../db/database.sqlite` должен отдавать **403/404** при корректной настройке Apache.

## Структура (кратко)

| Путь | Назначение |
|------|------------|
| `index.php` | Редирект на `dashboard` или `login` |
| `login.php` / `logout.php` | Сессия |
| `auth_check.php` | Проверка входа для страниц в `pages/` |
| `config.php` + `config.local.php` | Настройки (локальный файл не в git) |
| `app/` | БД, хелперы, layout, CSRF, бутстрап API |
| `db/schema.sql` | Схема SQLite |
| `db/seed.php` | Инициализация БД и admin-пользователь |
| `pages/*.php` | UI разделов |
| `api/*.php` | JSON (часть — с ключом) |
| `assets/` | Статичные `css`/`js` |
| `uploads/prices/` | Файлы прайсов (подкаталоги по месяцу) |
| `logs/app.log` | Опциональный лог (если каталог доступен на запись) |

---

## JSON API (кратко)

- **Публично:** `GET /api/health.php`
- **С ключом (или веб-сессия после входа):** `api/suppliers.php`, `api/products.php`, `api/price_uploads.php`, `api/leads.php`, `api/tasks.php`, `api/scenarios.php`, `api/prompts.php`
- **Заголовок:** `X-Api-Key: <значение из config>`, либо `Authorization: Bearer <тот_же_ключ>`.
- **Удаление** без метода `DELETE` в окружении: `POST` с `?op=delete&id=N` (или `action=delete`) / телом с `id`.
